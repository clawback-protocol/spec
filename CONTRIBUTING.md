   # Contributing to Clawback Protocol                                                                                  
                                                                                                                        
   Thanks for your interest in contributing. This project exists because people deserve real control over their data.   
                                                                                                                        
   ## How to Contribute                                                                                                 
                                                                                                                        
   ### Discussions & Ideas                                                                                              
   - Open an issue to propose changes to the spec                                                                       
   - Share use cases we haven't considered                                                                              
   - Point out security concerns or edge cases                                                                          
                                                                                                                        
   ### Code                                                                                                             
   - Fork the repo                                                                                                      
   - Create a feature branch (`git checkout -b feature/your-feature`)                                                   
   - Commit your changes (`git commit -m 'Add feature'`)                                                                
   - Push to your branch (`git push origin feature/your-feature`)                                                       
   - Open a Pull Request                                                                                                
                                                                                                                        
   ### Documentation                                                                                                    
   - Improve the spec clarity                                                                                           
   - Write integration guides                                                                                           
   - Fix typos (yes, really — they matter in a spec)                                                                    
                                                                                                                        
   ## Guidelines                                                                                                        
                                                                                                                        
   - **Security first.** If you find a vulnerability, please report it privately — do not open a public issue.          
   - **Keep it simple.** The protocol should be easy for developers to implement.                                       
   - **Explain your reasoning.** Spec changes should include rationale.                                                 
   - **Be respectful.** We're building something important. Keep discussions constructive.                              
                                                                                                                        
   ## Code of Conduct                                                                                                   
                                                                                                                        
   Be decent. Treat others how you'd want to be treated. We're here to protect people's privacy — let's start by        
 respecting each other.
